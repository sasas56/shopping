# 查询功能前后端工作过程说明

## 一、功能概述

本项目实现了商品查询和订单查询两大核心功能，采用前后端分离架构，通过 RESTful API 进行数据交互。

## 二、商品查询功能

### 2.1 功能需求

- 支持商品列表展示（分页）
- 支持按分类筛选
- 支持关键词搜索
- 支持按价格、销量排序

### 2.4 后端实现

#### 2.4.1 商品列表接口

**文件位置**：[server.js](../backend/server.js#L175-L210)

```javascript
app.get('/api/products', (req, res) => {
  const { 
    page = 1, 
    size = 10, 
    categoryId, 
    keyword,
    sortBy = 'createdAt',
    order = 'desc'
  } = req.query

  // 1. 构建筛选条件
  let filtered = [...products]

  // 2. 分类筛选
  if (categoryId) {
    filtered = filtered.filter(p => p.categoryId === Number(categoryId))
  }

  // 3. 关键词搜索
  if (keyword) {
    const kw = keyword.toLowerCase()
    filtered = filtered.filter(p => 
      p.name.toLowerCase().includes(kw) ||
      p.description.toLowerCase().includes(kw)
    )
  }

  // 4. 排序
  filtered.sort((a, b) => {
    const aVal = a[sortBy]
    const bVal = b[sortBy]
    return order === 'asc' ? aVal - bVal : bVal - aVal
  })

  // 5. 分页
  const total = filtered.length
  const start = (page - 1) * size
  const items = filtered.slice(start, start + size)

  // 6. 返回结果
  res.json({
    items,
    total,
    page: Number(page),
    size: Number(size)
  })
})
```

import type { Order, OrderListResponse } from '../types'

export interface OrderQuery {
  page?: number
  size?: number
  status?: string

export function getOrders(params: OrderQuery = {}) {
  return request.get<OrderListResponse>('/orders', { params })

export function getOrderDetail(id: number) {
  return request.get<Order>(`/orders/${id}`)
}

export function cancelOrder(id: number) {
  return request.put(`/orders/${id}/cancel`)
}

export function payOrder(id: number) {
  return request.put(`/orders/${id}/pay`)

export function confirmOrder(id: number) {
  return request.put(`/orders/${id}/confirm`)

#### 3.3.2 订单Store

**文件位置**：[order.ts](../src/stores/order.ts)

```typescript
import { defineStore } from 'pinia'
import { ref } from 'vue'
import { getOrders, getOrderDetail, cancelOrder, payOrder, confirmOrder } from '../api/orders'
import type { Order } from '../types'

export const useOrderStore = defineStore('order', () => {
  const orders = ref<Order[]>([])
  const currentOrder = ref<Order | null>(null)
  const total = ref(0)
  const loading = ref(false)

  async function fetchOrders(params = {}) {
    loading.value = true
    try {
      const result = await getOrders(params)
      orders.value = result.items
      total.value = result.total
    } finally {
      loading.value = false
    }
  }

  async function fetchOrderDetail(id: number) {
    currentOrder.value = await getOrderDetail(id)
  }

  async function cancelOrderAction(id: number) {
    await cancelOrder(id)
    await fetchOrders()
  }

  async function payOrderAction(id: number) {
    await payOrder(id)
    await fetchOrders()
  }

  async function confirmOrderAction(id: number) {
    await confirmOrder(id)
    await fetchOrders()
  }

  return {
    orders,
    currentOrder,
    total,
    loading,
    fetchOrders,
    fetchOrderDetail,
    cancelOrderAction,
    payOrderAction,
    confirmOrderAction
  }
})
```

### 3.4 后端实现

#### 3.4.1 订单列表接口

**文件位置**：[server.js](../backend/server.js#L459-L486)

```javascript
app.get('/api/orders', authenticateToken, (req, res) => {
  const { page = 1, size = 10, status } = req.query
  const userId = req.user.id

  // 1. 查询用户订单
  let userOrders = orders.filter(o => o.userId === userId)

  // 2. 状态筛选
  if (status) {
    userOrders = userOrders.filter(o => o.status === status)
  }

  // 3. 排序（按创建时间倒序）
  userOrders.sort((a, b) => new Date(b.createdAt) - new Date(a.createdAt))

  // 4. 分页
  const total = userOrders.length
  const start = (page - 1) * size
  const pagedOrders = userOrders.slice(start, start + size)

  // 5. 联表查询，补充商品和地址信息
  const enrichedOrders = pagedOrders.map(order => {
    const items = order.items.map(item => {
      const product = products.find(p => p.id === item.productId)
      return {
        ...item,
        name: product?.name,
        image: product?.image,
        price: product?.price
      }
    })

    const address = addresses.find(a => a.id === order.addressId)

    return {
      ...order,
      items,
      address
    }
  })

  // 6. 返回结果
  res.json({
    items: enrichedOrders,
    total,
    page: Number(page),
    size: Number(size)
  })
})
```

#### 3.4.2 订单详情接口

```javascript
app.get('/api/orders/:id', authenticateToken, (req, res) => {
  const orderId = Number(req.params.id)
  const order = orders.find(o => o.id === orderId)

  if (!order) {
    return res.status(404).json({ message: '订单不存在' })
  }

  if (order.userId !== req.user.id) {
    return res.status(403).json({ message: '无权查看此订单' })
  }

  // 补充商品和地址信息
  const items = order.items.map(item => {
    const product = products.find(p => p.id === item.productId)
    return {
      ...item,
      name: product?.name,
      image: product?.image,
      price: product?.price
    }
  })

  const address = addresses.find(a => a.id === order.addressId)

  res.json({
    ...order,
    items,
    address
  })
})
```

## 四、技术要点

### 4.1 前后端分离

- 前端：Vue3 + TypeScript + Pinia
- 后端：Node.js + Express + JWT
- 通信：RESTful API（JSON格式）

### 4.2 状态管理

- 使用 Pinia 管理全局状态
- 请求状态（loading）实时反馈
- 错误处理统一封装

### 4.3 性能优化

- 前端：路由懒加载、图片懒加载
- 后端：内存数据存储（Demo模式）
- 分页：避免大量数据传输

### 4.4 安全措施

- JWT Token 身份认证
- 路由守卫页面保护
- 接口权限验证
