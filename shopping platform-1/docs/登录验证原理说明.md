# 登录验证原理说明

## 一、技术选型

本项目采用 JWT (JSON Web Token) 实现用户身份验证，相比传统的 Session 认证具有以下优势：
- 无状态，服务器不需要存储会话信息
- 跨域支持，便于前后端分离架构
- 可扩展，便于后续接入多端

## 二、JWT 结构说明

JWT Token 由三部分组成，用 `.` 分隔：
```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.payload.signature
```

1. **Header**：声明类型和加密算法
2. **Payload**：存储用户信息（如用户ID、用户名、过期时间）
3. **Signature**：签名，防止篡改

       │ 存储Token到localStorage                    │ 包含用户信息

## 四、后端实现

### 4.1 登录接口

**文件位置**：[server.js](../backend/server.js#L143-L163)

```javascript
app.post('/api/users/login', (req, res) => {
  const { username, password } = req.body

  // 1. 验证用户是否存在
  const user = users.find(u => u.username === username)
  if (!user) {
    return res.status(401).json({ message: '用户名或密码错误' })
  }

  // 2. 验证密码
  if (user.password !== password) {
    return res.status(401).json({ message: '用户名或密码错误' })
  }

  // 3. 生成JWT Token
  const token = jwt.sign(
    { id: user.id, username: user.username },
    secretKey,
    { expiresIn: '24h' }
  )

  // 4. 返回Token和用户信息
  res.json({
    token,
    user: { id: user.id, username: user.username, ... }
  })
})
```

### 4.2 Token验证中间件

**文件位置**：[server.js](../backend/server.js#L113-L128)

```javascript
function authenticateToken(req, res, next) {
  // 1. 从请求头获取Token
  const authHeader = req.headers['authorization']
  const token = authHeader && authHeader.split(' ')[1]

  // 2. 检查Token是否存在
  if (!token) {
    return res.status(401).json({ message: 'Unauthorized' })
  }

  // 3. 验证Token有效性
  jwt.verify(token, secretKey, (err, user) => {
    if (err) {
      return res.status(403).json({ message: 'Invalid token' })
    }
    // 4. 将用户信息附加到请求对象
    req.user = user
    next()
  })
}
```

### 4.3 路由保护

受保护的接口需要在路由中添加 `authenticateToken` 中间件：

```javascript
app.get('/api/orders', authenticateToken, (req, res) => {
  // req.user 包含当前登录用户信息
  const orders = getOrdersByUserId(req.user.id)
  res.json({ items: orders })
})
```

## 五、前端实现

### 5.1 请求拦截器

**文件位置**：[request.ts](../src/utils/request.ts#L24-L35)

```typescript
// 请求拦截器：自动添加Token
instance.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

### 5.2 响应拦截器

**文件位置**：[request.ts](../src/utils/request.ts#L37-L86)

```typescript
// 响应拦截器：处理Token失效
instance.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) {
      // Token失效，清除本地存储并跳转登录页
      localStorage.removeItem('token')
      localStorage.removeItem('user')
      router.push({ name: 'login', query: { redirect: window.location.pathname } })
    }
    return Promise.reject(error)
  }
)
```

### 5.3 路由守卫

**文件位置**：[router/index.ts](../src/router/index.ts#L45-L60)

```typescript
router.beforeEach((to, _from, next) => {
  const token = localStorage.getItem('token')

  // 需要认证的页面，未登录则跳转登录
  if (to.meta.requiresAuth && !token) {
    ElMessage.warning('请先登录')
    next({ name: 'login', query: { redirect: to.fullPath } })
    return
  }

  // 已登录用户访问登录/注册页，跳转首页
  if (to.meta.guestOnly && token) {
    next({ name: 'home' })
    return
  }

  next()
})
```

## 六、安全措施

1. **Token有效期**：设置为24小时，过期需重新登录
2. **HTTPS传输**：生产环境必须使用HTTPS加密传输
3. **密码加密**：生产环境应对密码进行哈希存储（如bcrypt）
4. **敏感接口保护**：所有需要认证的接口都添加中间件验证
5. **错误处理**：统一错误响应格式，避免泄露敏感信息
